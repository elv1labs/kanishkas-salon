## Standard Deploy (after any code change)

```bash
cd /home/elv1/projects/kanishkas-salon

# 1. Pull latest code
git pull

# 2. Pre-deploy validation (catches type errors + missing env vars early)
bash scripts/pre-deploy-check.sh

# 3. Build
npm run build  # must exit 0 before proceeding

# 4. Copy static assets into standalone (not done automatically by next build)
cp -r .next/static  .next/standalone/.next/static
cp -r public/.      .next/standalone/public/

# 5. Ensure production env in standalone
sed -i 's/NODE_ENV=development/NODE_ENV=production/' .next/standalone/.env 2>/dev/null || true

# 6. Restart PM2
pm2 restart kanishkas-salon
pm2 save                    # persist in case of reboot

# 7. Verify
pm2 status                  # should show 0 restarts
curl -s http://localhost:3001/api/health | python3 -m json.tool
```

## Nginx Configuration

Save as `/etc/nginx/sites-available/kanishkas-salon`:

```nginx
server {
    listen 80;
    server_name kanishkassalon.com www.kanishkassalon.com 168.231.121.107;

    # Redirect HTTP → HTTPS (remove this block if not using SSL yet)
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name kanishkassalon.com www.kanishkassalon.com;

    # SSL — generated by Certbot (see SSL section below)
    ssl_certificate     /etc/letsencrypt/live/kanishkassalon.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/kanishkassalon.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    # Security
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Robots-Tag "index, follow" always;

    # Gzip
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml image/svg+xml;
    gzip_min_length 1024;

    # Static assets — long cache, served directly
    location /_next/static/ {
        alias /home/elv1/projects/kanishkas-salon/.next/standalone/.next/static/;
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    location /icons/ {
        alias /home/elv1/projects/kanishkas-salon/.next/standalone/public/icons/;
        expires 30d;
        add_header Cache-Control "public";
        access_log off;
    }

    location /uploads/ {
        alias /home/elv1/projects/kanishkas-salon/.next/standalone/public/uploads/;
        expires 7d;
        add_header Cache-Control "public";
    }

    # PWA files — short cache so updates propagate
    location ~ ^/(manifest\.json|sw\.js)$ {
        proxy_pass http://localhost:3001;
        expires 1h;
        add_header Cache-Control "public, max-age=3600";
    }

    # Proxy all other requests to Next.js
    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 60s;
        proxy_connect_timeout 60s;
        client_max_body_size 50M;  # allow file uploads
    }

    # Health check (no log spam)
    location = /api/health {
        proxy_pass http://localhost:3001;
        access_log off;
    }
}
```

Enable and test:
```bash
ln -s /etc/nginx/sites-available/kanishkas-salon /etc/nginx/sites-enabled/
nginx -t               # must print "test is successful"
systemctl reload nginx
```

## Daily Backup (PostgreSQL)

```bash
# /etc/cron.daily/salon-db-backup
#!/bin/bash
BACKUP_DIR="/var/backups/kanishkas-salon"
mkdir -p "$BACKUP_DIR"
pg_dump kanishkas_salon | gzip > "$BACKUP_DIR/$(date +%Y%m%d).sql.gz"
# Keep last 14 days
find "$BACKUP_DIR" -mtime +14 -delete
```

## Restore from Backup (PostgreSQL)

```bash
# 1. List available backups
ls -lh /var/backups/kanishkas-salon/

# 2. Stop the app to prevent writes during restore
pm2 stop kanishkas-salon

# 3. Restore from a specific backup file
gunzip -c /var/backups/kanishkas-salon/20260410.sql.gz | psql kanishkas_salon

# If you need to drop and recreate the database first (DESTRUCTIVE):
# dropdb kanishkas_salon
# createdb kanishkas_salon
# gunzip -c /var/backups/kanishkas-salon/20260410.sql.gz | psql kanishkas_salon

# 4. Verify the restore
psql kanishkas_salon -c "SELECT COUNT(*) FROM \"User\";"

# 5. Restart the app
pm2 start kanishkas-salon
pm2 save
```

> ⚠️ Restoring will **overwrite** all current data in the target database. Always confirm you're restoring the correct backup file. If restoring to a fresh database, use the `dropdb`/`createdb` approach.

## 💬 ngrok Tunnel Management

The ngrok tunnel at `https://speculatively-fictional-joni.ngrok-free.dev` is used as a temporary trusted HTTPS URL for PWA installation. It must be kept running for the PWA to work as a standalone app.

```bash
# Check if ngrok is running
curl -s http://localhost:4040/api/tunnels | python3 -m json.tool
# or
curl -s http://localhost:4041/api/tunnels | python3 -m json.tool

# If ngrok has stopped, restart it
ngrok http 3001 &

# Get the new URL and update public/docs/install-qr.html line 414
```

> ⚠️ ngrok free tier generates a new URL on each restart. Always update the QR code file and resend to users after restarting ngrok.
